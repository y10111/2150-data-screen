# 热轧2150线数据化大屏运营决策平台 - 开发需求说明

---

## 一、项目概述

本项目旨在为热轧带钢厂2150线构建一个数据化大屏运营决策平台，通过可视化大屏实时监控产线关键指标（KPI），并融入人工智能（AI）能力，实现异常预警、根因分析、优化建议等智能决策支持。平台基于 Vue2 + ECharts + Mock.js 开发，开发阶段使用 Mock 数据模拟真实业务场景，后期可平滑对接各业务部门真实数据接口。

**核心目标**：
- 将分散在 Excel、PPT 中的生产、质量、设备、库存等数据集中展示，实现产线状态“一屏尽览”。
- 利用 AI 模拟技术，提供预警、根因分析、趋势预测等功能，辅助管理人员快速决策。
- 构建可复用的数据大屏框架，为后续扩展至其他产线奠定基础。

## 三、整体布局结构

页面采用固定宽高比（16:9），优先适配 1920×1080 分辨率，整体分为五个区域：

```
+--------------------------------------------------+
|                    Header                        |
+--------------------------------------------------+
|                 KPI 卡片行（8个）                 |
+--------------------------------------------------+
|     左侧区域     |       中央区域       |   右侧区域   |
|   (设备+质量)    |   (产量核心大图)     | (能耗+库存)  |
|                  |                      |             |
|                  |      跨两行显示       |             |
|                  |                      |             |
+--------------------------------------------------+
|                 第四行（AI辅助区）                |
+--------------------------------------------------+
|                    Footer                        |
+--------------------------------------------------+
```

**说明**：
- 中央区域跨第二行和第三行，占据最大视觉面积，用于展示产量趋势核心图。
- 左侧区域和右侧区域各占一列，内容上下排列，分别展示设备+质量、能耗+库存相关数据。
- 第四行为独立的 AI 辅助区，集中展示预警与建议。

## 四、各区域详细设计

### 1. Header
- **左侧**：系统名称“热轧2150线数据化大屏运营决策平台”
- **右侧**：当前时间（实时刷新）、当班信息（白班/夜班）、数据更新时间戳、全屏切换按钮
- **样式**：深色背景，字体白色，高度 60px

### 2. KPI 卡片行（第一行）
展示 8 个核心指标卡片，每个卡片包含：
- 指标名称
- 当前数值（保留两位小数）
- 单位
- 环比趋势（上升/下降箭头）
- 预警状态（超出阈值时卡片背景变色并显示预警图标）

| 指标 | 示例值 | 单位 | 预警阈值 | 数据来源字段 |
|------|--------|------|----------|--------------|
| 当日产量 | 14276 | 吨 | <计划95% | `today_actual` |
| 计划完成率 | 99.6 | % | <95% | `completion_rate` |
| 成材率 | 98.61 | % | <98% | `yield_rate` |
| 热装率 | 52.42 | % | <50% | `hot_charge_rate` |
| 煤气单耗 | 1.24 | GJ/t | >1.30 | `gas_consumption` |
| 质量封锁 | 76 | 卷 | >50 | `quality_lock` |
| 设备事故时间 | 2.09 | 小时 | >1 | `device_downtime` |
| 钢坯库存 | 0.2739 | 万吨 | <0.25 | `inventory` |

**交互**：鼠标悬浮卡片显示指标定义；点击卡片可高亮关联图表（如点击“质量封锁”，右侧图表高亮封锁趋势）。

### 3. 左侧区域（设备+质量）
此区域分为上下两个模块。

#### 3.1 上半部分：设备与事故分析
- **图表类型**：折线图 + 堆叠柱状图
- **横轴**：日期（3月1日 - 3月23日）
- **纵轴**：设备事故时间（小时）
- **数据内容**：
  - 折线：每日设备事故总时间
  - 堆叠柱状：按事故类型（电气、机械、其他）分层展示
- **数据来源**：调度图表“机组停轧原因分析”中的“合计”“机械”“电气”列
- **交互**：点击某柱可查看当日事故明细（弹窗）

#### 3.2 下半部分：质量与成材分析
- **图表类型**：柱状图 + 折线图
- **横轴**：日期（3月1日 - 3月23日）
- **纵轴左**：质量封锁卷数（柱状）
- **纵轴右**：成材率%（折线）
- **数据来源**：质量封锁统计 + 调度图表“轧线成材率%”
- **交互**：鼠标悬停显示具体数值

### 4. 中央区域（产量核心大图）
- **占据第二行和第三行**，高度约为 600px
- **图表类型**：面积图 + 柱状图组合
- **横轴**：日期（3月1日 - 3月23日）
- **纵轴**：产量（吨）
- **数据内容**：
  - 柱状图：每日实轧产量
  - 面积填充：计划产量（虚线或半透明填充），欠产区域用红色半透明标出
- **增强元素**：
  - 图表上方标注累计欠产（-18067吨）和最高/最低产量点
  - 右侧小仪表盘显示累计欠产占比
  - 鼠标悬停显示该日计划产量、实轧产量、欠产原因（可从历史欠产原因中匹配）
  - 支持点击某日，联动下方 AI 面板显示该日详细分析
- **数据来源**：调度图表“轧出量”合计（计划与实轧）

### 5. 右侧区域（能耗+库存）
此区域分为上下两个模块。

#### 5.1 上半部分：能耗与热装分析
- **图表类型**：双轴图
- **横轴**：日期（3月1日 - 3月23日）
- **纵轴左**：热装率（%）
- **纵轴右**：煤气单耗（GJ/t）
- **数据内容**：
  - 折线1：热装率变化
  - 折线2：煤气单耗变化
- **数据来源**：调度图表“热装率%”和“煤气单耗”
- **交互**：鼠标悬停显示具体数值，可高亮关联点

#### 5.2 下半部分：库存与缴库分析
- **图表类型**：面积图 + 柱状图
- **横轴**：日期（3月1日 - 3月23日）
- **纵轴左**：钢坯库存（万吨，面积图）
- **纵轴右**：缴库量（吨，柱状图）
- **数据来源**：钢坯库存走势图 + 调度图表“缴库量”
- **交互**：鼠标悬停显示数值

### 6. 第四行（AI辅助区）
- **整行高度**：200-250px
- **内容**：AI 智能预警与建议面板（卡片或滚动列表形式）
- **展示信息**：
  - **实时预警**：当前产线异常（如“当前产量落后计划8%，建议检查卷取机状态”）
  - **根因分析**：根据异常自动推断可能原因（如“近期欠产主要与待热时间增加有关，建议优化加热炉节奏”）
  - **优化建议**：如“热装率偏低，预计明日煤气单耗将上升0.05 GJ/t，建议调整装炉节奏”
  - **预测信息**：明日产量预测、库存预警
  - **RPA状态**：今日调度会汇报已自动生成
- **数据来源**：前端模拟逻辑（基于历史数据和规则），后期对接 AI 引擎
- **交互**：点击建议可查看详情或跳转至相关图表

### 7. Footer
- **左侧**：数据更新时间
- **中间**：数据来源说明（如“数据基于调度图表、质量封锁统计等”）
- **右侧**：技术支持、版本号
- **样式**：深色背景，高度 40px

## 五、AI集成点设计（模拟）

虽然开发阶段使用 Mock 数据，但需预留 AI 接口，并在前端模拟以下功能：

1. **实时预警**
   - 在 KPI 卡片上，当数值超出阈值时卡片背景变浅红，并显示预警图标
   - 在 AI 辅助区推送预警消息（如“热装率低于50%”）

2. **根因分析**
   - 当产量欠产时，AI 辅助区自动显示“可能原因：卷取故障/待热时间长”
   - 基于历史“欠产原因”字段的关键词匹配模拟

3. **产量预测**
   - 在中央区域或 AI 辅助区显示“明日产量预测：14400-14600吨”
   - 基于历史产量移动平均模拟

4. **库存预警**
   - 当库存低于安全线时，推送“建议增加热装量或加快外发”

5. **自然语言交互（加分）**
   - 在 Header 增加搜索框，支持简单提问如“最近三天质量封锁情况”，前端通过关键词匹配高亮对应图表

## 六、数据接口需求（Mock设计）

为支持以上图表，前端需要以下接口（Mock.js 拦截模拟）：

| 接口 | 方法 | 返回数据说明 |
|------|------|--------------|
| `/api/dashboard/kpi` | GET | 实时 KPI 数据（今日累计产量、完成率、成材率等） |
| `/api/dashboard/trend` | GET | 每日趋势数据（数组，包含日期、计划、实轧、成材率、热装率、煤气单耗、封锁卷数、设备事故时间、事故类型、库存、缴库量） |
| `/api/dashboard/inventory` | GET | 库存历史数据（日期、库存量） |
| `/api/dashboard/defect` | GET | 当日缺陷分布（卷形、板形等） |
| `/api/dashboard/ai/alerts` | GET | AI 预警和建议列表（文本、类型、时间） |
| `/api/dashboard/device/health` | GET | 关键设备健康评分（设备名、评分） |

**Mock 数据设计原则**：基于真实历史数据（3月1日 - 3月23日）生成合理趋势，数值在合理范围内波动，模拟阈值触发情况。具体 Mock 数据详见《热轧2150线数据化大屏运营决策平台 - 完整Mock数据定义》。

## 七、交互与体验要求

- **响应式**：设计为固定比例（16:9），优先适配 1920×1080 分辨率，保证在常用大屏上正常显示。
- **动画**：图表加载时有渐变动画，数值变化时数字滚动。
- **联动**：点击中央产量图的某一天，可联动刷新 AI 面板显示该日分析。
- **自动刷新**：每 10 秒刷新 KPI 数据和预警信息（模拟实时数据）。
- **加载状态**：数据加载时显示骨架屏或 loading 动画。

## 八、开发注意事项

1. **项目结构**：使用 Vue CLI 创建项目，组件化开发，按区域拆分组件（Header、KPICard、LeftPanel、CenterChart、RightPanel、AIAssistant、Footer）。
2. **ECharts 主题**：统一使用暗色主题（适配工业大屏风格），全局配置，可参考 ECharts 官方暗色主题或自定义。
3. **Mock 开关**：通过环境变量 `VUE_APP_USE_MOCK` 控制是否启用 Mock，方便后期对接真实接口。
4. **代码规范**：遵循 ESLint + Prettier，提交前进行代码格式化。
5. **性能优化**：
   - ECharts 实例复用，避免重复创建。
   - 数据请求使用防抖（如自动刷新）。
   - 大屏组件按需加载。
