# 热轧2150线数据化大屏运营决策平台 - 项目搭建方案（更新版）

---

## 一、环境要求

- **Node.js**：v14.x 或更高版本（推荐 v16 LTS）
- **包管理器**：npm 或 yarn（推荐 yarn）
- **开发工具**：VS Code（推荐安装 Vetur、ESLint、Prettier 插件）

---

## 二、创建项目

使用 Vue CLI 创建 Vue2 项目：

```bash
# 全局安装 Vue CLI（如未安装）
npm install -g @vue/cli

# 创建项目
vue create 2150-data-screen

# 选择手动配置
# - 选择 Vue 2.x
# - 勾选 Router、Vuex、CSS Pre-processors、Linter/Formatter
# - 使用 history 模式（或默认 hash）
# - 选择 Less/Sass（推荐 Less）
# - 选择 ESLint + Prettier
```

---

## 三、安装核心依赖

进入项目目录，安装以下依赖：

```bash
# 基础框架与UI
npm install element-ui @jiaminghi/data-view axios echarts mockjs dayjs lodash

# 开发依赖（如果选择 Less）
npm install -D less less-loader

# 可选：更好的 ECharts 封装
npm install vue-echarts
```

**依赖说明**：

| 依赖 | 用途 |
|------|------|
| element-ui | 基础 UI 组件（卡片、布局、按钮等） |
| @jiaminghi/data-view | 数据大屏专用组件（边框、装饰、数字翻牌器等） |
| axios | 数据请求 |
| echarts | 图表可视化 |
| mockjs | 模拟数据（开发阶段） |
| dayjs | 时间格式化 |
| lodash | 工具函数（防抖、深拷贝等） |
| less | CSS 预处理器（样式编写） |
| vue-echarts | ECharts 的 Vue 封装（可选） |

## 七、Mock 数据配置

### 1. 环境变量

创建 `.env.development` 文件：

```
VUE_APP_USE_MOCK=true
```

`.env.production`：

```
VUE_APP_USE_MOCK=false
```

### 2. Mock 入口文件

在 `src/mock/index.js` 中：

```javascript
import Mock from 'mockjs'
import kpiData from './data/kpi.json'
import trendData from './data/trend.json'
import inventoryData from './data/inventory.json'
import defectData from './data/defect.json'
import aiAlertsData from './data/aiAlerts.json'
import deviceHealthData from './data/deviceHealth.json'

Mock.setup({ timeout: '200-600' })

Mock.mock('/api/dashboard/kpi', 'get', {
  code: 200,
  data: kpiData
})

Mock.mock('/api/dashboard/trend', 'get', {
  code: 200,
  data: trendData
})

Mock.mock('/api/dashboard/inventory', 'get', {
  code: 200,
  data: inventoryData
})

Mock.mock('/api/dashboard/defect', 'get', {
  code: 200,
  data: defectData
})

Mock.mock('/api/dashboard/ai/alerts', 'get', {
  code: 200,
  data: aiAlertsData
})

Mock.mock('/api/dashboard/device/health', 'get', {
  code: 200,
  data: deviceHealthData
})
```

### 3. 在 `main.js` 中引入 Mock

```javascript
import Vue from 'vue'
import App from './App.vue'
import router from './router'
import store from './store'
import ElementUI from 'element-ui'
import 'element-ui/lib/theme-chalk/index.css'
import datav from '@jiaminghi/data-view'
import './styles/global.less'

Vue.use(ElementUI)
Vue.use(datav)

// 开发环境下启用 Mock
if (process.env.NODE_ENV === 'development' && process.env.VUE_APP_USE_MOCK === 'true') {
  require('./mock')
}

Vue.config.productionTip = false

new Vue({
  router,
  store,
  render: h => h(App)
}).$mount('#app')
```

## 八、axios 封装（utils/request.js）

```javascript
import axios from 'axios'

const service = axios.create({
  baseURL: process.env.VUE_APP_BASE_API || '/api',
  timeout: 15000
})

// 请求拦截器
service.interceptors.request.use(
  config => {
    // 添加 token 等
    return config
  },
  error => Promise.reject(error)
)

// 响应拦截器
service.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code !== 200) {
      return Promise.reject(new Error(res.message || 'Error'))
    }
    return res.data
  },
  error => Promise.reject(error)
)

export default service
```

## 十一、组件化开发建议

### 1. 页面主组件（views/Dashboard.vue）

使用 Element UI 的 `el-row` 和 `el-col` 配合 DataV 边框组件实现布局：

```vue
<template>
  <div class="dashboard">
    <Header />
    <el-row :gutter="16" class="kpi-row">
      <el-col :span="3" v-for="kpi in kpiList" :key="kpi.name">
        <KPICard :data="kpi" />
      </el-col>
    </el-row>
    <div class="main-area">
      <dv-border-box-1 class="left-panel">
        <LeftPanel :deviceData="deviceData" :qualityData="qualityData" />
      </dv-border-box-1>
      <dv-border-box-2 class="center-panel">
        <CenterChart :productionData="productionData" />
      </dv-border-box-2>
      <dv-border-box-3 class="right-panel">
        <RightPanel :energyData="energyData" :inventoryData="inventoryData" />
      </dv-border-box-3>
    </div>
    <AIAssistant :alerts="aiAlerts" />
    <Footer />
  </div>
</template>
```

### 2. 图表组件复用

封装一个 `BaseChart` 组件，接收 `option` 和 `theme`，内部使用 `vue-echarts` 或原生 ECharts。

## 十三、后期对接真实接口

1. **修改环境变量**：将 `.env.development` 中的 `VUE_APP_USE_MOCK` 设为 `false`，并配置 `VUE_APP_BASE_API` 为真实接口地址。
2. **删除或注释** `main.js` 中的 `require('./mock')` 引入。
3. **调整 axios baseURL**：确保 `request.js` 中的 `baseURL` 指向真实服务。
4. **处理接口字段映射**：如果真实接口返回字段与 Mock 不一致，可在 `api` 层做一层适配。
