# 热轧2150线数据化大屏运营决策平台 - AI模块需求分析与实现方案

---

## 一、AI模块定位与目标

### 1.1 定位
AI模块是数据化大屏的“智慧大脑”，通过统计分析、机器学习、运筹优化等技术，对产线运营数据进行深度挖掘，实现：

- **异常实时感知**：自动发现生产异常并预警。
- **根因智能诊断**：快速定位问题根源。
- **趋势前瞻预测**：预判未来关键指标走势。
- **决策优化建议**：提供可量化的改进措施。

### 1.2 核心目标
- 将传统的事后报表分析升级为实时智能决策。
- 降低管理人员数据分析门槛，提升响应速度。
- 通过量化预测和优化，辅助降低运营成本、提高效率。

## 三、AI功能详细设计

### 3.1 实时异常检测

**目标**：自动识别KPI异常波动，及时预警。

**方法**：
- **动态阈值法**：基于历史数据（最近7天）计算均值和标准差，设定报警区间 `[μ-3σ, μ+3σ]`。当日数值超出区间即触发预警。
- **环比异常**：计算当日值较昨日变化率，超过预设阈值（如±20%）触发预警。

**预警分级**：
- **红色（严重）**：超出3σ区间或设备停机、质量封锁突增。
- **黄色（警告）**：超出2σ区间或趋势恶化。
- **蓝色（提示）**：环比变化显著但仍在正常范围。

**输出示例**：
- 热装率52.4%（目标≥55%），低于正常区间下限，触发黄色预警。

### 3.2 根因分析

**目标**：当异常发生时，自动推断可能原因。

**方法**：
- **关联规则挖掘**：使用Apriori算法，从历史数据中提取“异常事件→可能原因”的规则。例如：`{欠产, 设备事故时间>2h} → {卷取机故障}`。
- **决策树分类**：以异常指标为目标变量，其他生产参数为特征，训练决策树模型，输出关键影响因素及其权重。

**输出示例**：
- 今日产量欠产，可能原因：卷取机故障（置信度85%）、待热时间增加（65%）。

### 3.3 趋势预测

**目标**：预测未来1-3天的关键指标，辅助提前决策。

**方法**：
- **时间序列模型**：
  - 对产量、质量封锁、库存等指标，使用ARIMA或Prophet进行预测。
  - 对成材率、热装率等相对稳定的指标，采用移动平均或指数平滑。
- **机器学习回归**：对于产量预测，可加入计划产量、设备状态等特征，使用XGBoost回归模型。

**输出示例**：
- 明日产量预测：14400~14600吨，概率90%。
- 库存预计3天后降至2.0万吨，低于安全线2.5万吨。

### 3.4 优化建议

**目标**：基于数据分析，给出量化可执行的优化措施。

**方法**：
- **回归分析**：建立热装率与煤气单耗的回归模型，量化热装率提升对能耗的节约效果。
- **线性规划**：以库存成本最小化为目标，求解最优补货/消耗策略。
- **仿真模拟**：基于历史数据，模拟不同措施的效果（如增加热装量、优化换辊周期）。

**输出示例**：
- 热装率提升至55%，预计煤气单耗可降至1.18 GJ/t，每日节约成本约1.2万元。
- 建议优先处理卷取机变频柜问题，可减少非计划停机2小时/周。

### 3.5 自然语言交互（加分项）

**目标**：支持通过自然语言查询数据和分析结果。

**方法**：
- **关键词匹配**：前端解析用户输入，匹配预设的查询意图（如“最近三天质量封锁”“产量趋势”），调用对应图表或数据。
- **简单对话模型**：可对接大语言模型API（如智谱AI、百度文心），实现复杂问答。

**示例交互**：
- 用户输入：“最近三天质量封锁情况？” → 系统高亮显示最近三天质量封锁柱状图，并在AI面板展示分析：“3月21-23日质量封锁分别为57、61、76卷，呈上升趋势，建议关注卷取工序。”

## 四、模型算法选择与实现路径

| 功能 | 推荐算法 | 实现难度 | 备注 |
|------|----------|----------|------|
| 异常检测 | 3-sigma、孤立森林 | 低 | 开发阶段用规则模拟 |
| 根因分析 | Apriori、决策树 | 中 | 需历史数据标签 |
| 产量预测 | ARIMA、XGBoost | 中 | 适合周期性数据 |
| 库存预警 | 时间序列预测+安全库存 | 低 | 可用移动平均 |
| 能耗优化 | 线性回归 | 低 | 直接关系明显 |
| 自然语言交互 | 关键词匹配、LLM API | 中 | 可选 |

**开发路径**：
1. **阶段一（模拟）**：所有AI功能使用前端规则模拟（阈值判断、关键词匹配、移动平均等），确保演示效果。
2. **阶段二（模型训练）**：收集更多历史数据，离线训练模型，导出模型文件。
3. **阶段三（接口部署）**：将模型封装为Python API服务，前端调用真实接口。

## 六、后期对接真实AI引擎的接口设计

当准备接入真实AI模型时，前端需预留接口，后端提供以下API：

| 接口 | 方法 | 请求参数 | 返回 |
|------|------|----------|------|
| `/api/ai/anomaly` | POST | `{metric, value, date}` | `{is_anomaly, severity, message}` |
| `/api/ai/root_cause` | POST | `{anomaly_type, date}` | `{causes: [{reason, confidence}]}` |
| `/api/ai/forecast` | POST | `{metric, days}` | `{forecast: [{date, low, high, median}]}` |
| `/api/ai/suggestion` | POST | `{context}` | `{suggestions: [{title, message, impact}]}` |
| `/api/ai/chat` | POST | `{query}` | `{answer, related_charts}` |

前端只需将参数发送给对应接口，并渲染返回结果，完全解耦AI实现。

## 七、可视化呈现与交互设计

### 7.1 AI模块布局（第四行）
- **左侧**：异常检测与根因分析（列表卡片）

- **实时刷新**：每10秒调用异常检测接口，更新预警列表。
- **点击查看详情**：点击预警条目，弹出详细分析弹窗（包含历史趋势、影响因素排行）。
- **图表联动**：点击AI建议中的“查看关联图表”，自动跳转到对应图表并高亮。

### 7.3 视觉风格
- 预警卡片根据严重程度使用不同边框颜色（红/黄/蓝）。
- 建议卡片显示量化影响（如“预计节约1.2万元/日”）。
